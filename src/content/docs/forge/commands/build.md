---
title: "op-forge build"
description: "Compile AssemblyScript contracts to WASM bytecode."
---

Compile your AssemblyScript contract source files into WASM bytecode ready for deployment on OPNet.

## Usage

```bash
op-forge build [options]
```

## Options

| Option           | Description                          | Default           |
| ---------------- | ------------------------------------ | ----------------- |
| `--root <path>`  | Project root directory               | Current directory |
| `--out <dir>`    | Output directory for compiled files  | `out`             |
| `--sizes`        | Print file sizes after compilation   | `false`           |

## Examples

Build from the project root:

```bash
op-forge build
```

Build with size reporting:

```bash
op-forge build --sizes
```

Build from a different directory:

```bash
op-forge build --root /path/to/project
```

Output to a custom directory:

```bash
op-forge build --out artifacts
```

## Output

The build produces two files per contract in the output directory:

| File             | Description                                  |
| ---------------- | -------------------------------------------- |
| `Contract.wasm`  | Compiled WebAssembly bytecode for deployment |
| `Contract.wat`   | Human-readable WebAssembly text format       |

Example:

```
out/
  Counter.wasm    # Deploy this
  Counter.wat     # Human-readable, useful for debugging
```

## Compiler Configuration

The build uses `asconfig.json` for AssemblyScript compiler settings. This file is generated by `op-forge init` and typically does not need manual editing.

Key compiler settings are handled automatically:

- Target: WASM
- Runtime: OPNet btc-runtime
- Optimization: Controlled via `opnet.toml` (`optimizer` and `optimizer_runs`)

## Errors

Common build errors:

**Missing dependencies** -- Run `pnpm install` before building.

```
Error: Cannot find module '@btc-vision/btc-runtime/runtime'
```

**Type errors** -- Fix TypeScript/AssemblyScript type issues in your source code.

```
Error: Type 'string' is not assignable to type 'u256'
```

**Missing asconfig.json** -- Ensure the file exists in your project root. Re-run `op-forge init` if needed.
