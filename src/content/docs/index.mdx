## Quick Start

Initialize a new project, build it, test it, and deploy it -- all from the command line.

```bash
# Scaffold a new project
op-forge init my-contract

# Build
cd my-contract
pnpm install
op-forge build

# Test
op-forge test

# Deploy to testnet
op-forge create MyContract --network testnet --private-key <WIF>
```

## Why Foundry?

OPNet Foundry is designed for developers building smart contracts on Bitcoin via the OPNet protocol. It handles the full lifecycle from project scaffolding to deployment.

**Familiar patterns.** If you have used Foundry for Solidity (forge, cast, anvil, chisel), the command structure will feel familiar. OPNet Foundry adapts the same workflow for AssemblyScript contracts targeting Bitcoin.

**One toolchain.** No need to stitch together separate compilers, test runners, and deployment scripts. op-forge handles compilation, testing, gas snapshots, artifact inspection, and deployment in a single CLI.

**Bitcoin-native.** Contracts compile to WASM, deploy via Bitcoin transactions, and interact through the OPNet protocol. Fee rates, UTXO management, and simulation are built in.
